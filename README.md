# 🔪 The Crime — لعبة لغز الجريمة

A multiplayer mystery game where players work together to uncover the criminal among them. The game is powered by **Google Gemini AI** which dynamically generates unique crime scenarios, character profiles, tangible evidence, and testimonies — all in **Egyptian Arabic dialect (العامية المصرية)**.

## Overview

**The Crime** is a social deduction game for 3–8 players. One (or more) players are secretly assigned as criminals, and the rest are innocents. Each round, players receive partial tangible evidence, share testimonies, discuss, and vote to eliminate who they think the criminal is.

The twist: **every game scenario is uniquely generated by AI in 3 sequential steps**, so no two games are alike.

## 3-Step AI Pipeline

The scenario is generated through **3 reactive Gemini API calls** (using `flatMap` chains), each building upon the previous response:

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: FOUNDATION — Full Narrative Story                  │
│  Input:  Player names, setting, criminal count              │
│  Output: fullNarrative, players{}, crimeBriefing,           │
│          groundTruth, master_timeline, setting              │
│                                                             │
│  • Complete crime story in chronological order              │
│  • Character profiles with roles, alibis, motives           │
│  • Players returned as JSON object with real names as keys  │
└──────────────────────┬──────────────────────────────────────┘
                       │ Foundation JSON passed to Step 2
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 2: CLUES — Tangible Evidence Extraction               │
│  Input:  fullNarrative from Step 1 + player names           │
│  Output: {clues: [...]}                                     │
│                                                             │
│  • 3-5 tangible physical evidence items                     │
│  • Types: CRIMINAL, SUSPICION, ALIBI, RED_HERRING           │
│  • Each clue has: holder, type, clue text, targets,         │
│    hook_player, hook_sentence, chain_connects_to            │
│  • Holder names must be real player names                   │
└──────────────────────┬──────────────────────────────────────┘
                       │ Foundation + Clues passed to Step 3
                       ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 3: TESTIMONIES — Hook-Based Witness Statements        │
│  Input:  fullNarrative + clues JSON                         │
│  Output: {testimonies: {"PlayerName": "statement..."}}      │
│                                                             │
│  • 3-4 sentence first-person testimony per player           │
│  • Must contain hook_sentences from Step 2                  │
│  • Each testimony has alibi + contradiction + relationship  │
└─────────────────────────────────────────────────────────────┘
```

### Why 3 Steps?

| Problem with single prompt          | 3-Step Solution                                            |
| ------------------------------------ | ---------------------------------------------------------- |
| Clues unrelated to the story         | Step 2 extracts clues FROM the story — grounded generation |
| Hook sentences missing in testimony  | Step 3 receives exact hooks and embeds them                |
| Generic player references            | Each step enforces real player names                       |
| Inconsistent evidence types          | Step 2 has strict type classification rules                |
| Overwhelming single prompt           | Each step is focused and manageable                        |

## Prompt Engineering Deep Dive

### System Prompt — The Game Master Persona

The system prompt establishes the AI as a crime mystery game master in the style of Agatha Christie and Sherlock Holmes. Key rules enforced:

| Rule                            | Details                                                                     |
| ------------------------------- | --------------------------------------------------------------------------- |
| **Egyptian Arabic only**        | عامية مصرية — "شاف" not "رأى", "راح" not "ذهب"                             |
| **Everyone looks suspicious**   | Every innocent has a real motive + personal secret                          |
| **Dynamic round progression**   | Rounds scale with player count: innocents + 1 (criminal reveal last)       |
| **Tangible physical evidence**  | Receipts, fabric, fingerprints, mud — not abstract observations            |
| **Alibi puzzle**                | Criminal has strong alibi with one hidden crack                            |
| **No smoking guns**             | No single clue reveals the criminal alone                                  |

### Foundation Prompt (Step 1) — Key Features

- Enforces JSON **object** format for players with real names as keys
- Requires `fullNarrative`, `master_timeline`, `crimeBriefing`, `groundTruth`
- Each innocent gets: motive, alibi, personal secret, location, blind spot
- Each criminal gets: cover story, alibi crack, tactical note

### Clues Prompt (Step 2) — Tangible Evidence

- **3-5 physical evidence items** extracted from the story
- Examples: مطعم receipt, mud on shoes, burnt text message, fabric piece, lipstick mark
- Distribution rules prevent all clues pointing at one person
- `holder` field uses **real player names** (enforced in prompt with ⚠️ warnings)
- Hook system: each clue links to a testimony sentence

### Testimonies Prompt (Step 3) — Witness Statements

- Each testimony includes alibi + relationship + subtle contradiction
- Must copy-paste `hook_sentences` from Step 2
- Criminal testimony has convincing cover story with hidden flaw

### AI Configuration

```yaml
model: gemini-2.5-flash
temperature: 0.5      # Balanced creativity for structured output
topK: 40              # Broad token sampling
topP: 0.95            # Nucleus sampling
maxOutputTokens: 65536 # Large enough for 3-step pipeline
responseMimeType: application/json  # Force JSON output
```

## AI Response Parsing

After all 3 Gemini calls complete, the backend:

1. **Logs all 3 raw JSON responses** for debugging
2. **Handles both JSON formats** — players as object (preferred) or array (fallback with index mapping)
3. **Fuzzy name matching** — clue `holder` matched via contains-based logic for Arabic name variations
4. **Assigns roles** — criminal or innocent per AI's decision
5. **Distributes packages** — each player gets their private info:
   - Character description
   - Mandatory testimony (from Step 3)
   - Private clues (tangible evidence from Step 2, revealed one per round)
   - Criminal extras: cover story, tactical note, knowledge about others
   - Innocent extras: location, blind spot
6. **Validates hooks** — checks that hook_sentences appear in the correct testimony
7. **Stores ground truth** — only revealed when the game ends

### Criminal vs. Innocent Information Flow

```
     Step 1: Foundation (narrative + characters)
                    │
     Step 2: Clues (tangible evidence)
                    │
     Step 3: Testimonies (hook-based statements)
                    │
          Backend assembles packages
                    │
     ┌──────────────┴──────────────┐
     ▼                             ▼
  INNOCENT                      CRIMINAL
  Gets real tangible clues      Gets RED_HERRING clues
  (physical evidence,           (fabricated evidence with
   first-person,                 ⚠️ warning prefix, designed
   from the story)               to frame innocent players)
                                Knows fellow criminals
  Can piece together            Must maintain cover story
  truth with others             Testimony has a hidden flaw
```

## Win Conditions

**Innocents win** when all criminals are eliminated.

**Criminals win** when:
- Active innocents ≤ active criminals (criminals equal or outnumber innocents), **or**
- Maximum rounds are reached without catching all criminals

## How to Run

### Prerequisites

- Java 17+
- Node.js 18+
- A [Google Gemini API key](https://makersuite.google.com/app/apikey)

### 1. Configure the API Key

Open `backend/src/main/resources/application.yml` and set:

```yaml
gemini:
  api-key: YOUR_GEMINI_API_KEY_HERE
  model: gemini-2.5-flash
```

> ⚠️ **See [API Key Security](#api-key-security) before pushing to GitHub.**

### 2. Start the Backend

```bash
cd backend
./mvnw spring-boot:run
```

The backend runs on **http://localhost:8080**.

> If port 8080 is already in use: `kill $(lsof -ti:8080)` then retry.

### 3. Start the Frontend

```bash
cd frontend
npm install
npm run dev
```

The frontend runs on **http://localhost:3000**.

### 4. Play

1. Open **http://localhost:3000**
2. Create a room (set your name, optional setting, criminal count 1–3)
3. Share the 6-character room code with friends
4. Friends join using the code
5. Host starts the game — AI generates the scenario (~20–40 seconds for 3 API calls)
6. Read the crime briefing together
7. Check your private role card — you'll see your character, testimony, and first clue
8. Each round: get a new tangible clue → discuss out loud → vote
9. Eliminate suspects until the criminal is caught or the criminals win!

## API Key Security

The API key is currently stored in `application.yml`. **Do not commit your real key to a public repository.**

**Option A — Environment variable (recommended):**

```yaml
# application.yml
gemini:
  api-key: ${GEMINI_API_KEY}
```

```bash
export GEMINI_API_KEY=your_key_here
mvn spring-boot:run
```

**Option B — Git-ignored local override file:**

Create `backend/src/main/resources/application-local.yml` (already in `.gitignore`):

```yaml
gemini:
  api-key: your_key_here
```

```bash
mvn spring-boot:run -Dspring.profiles.active=local
```
