# 🔪 The Crime — لعبة لغز الجريمة

A multiplayer mystery game where players work together to uncover the criminal among them. The game is powered by **Google Gemini AI** which dynamically generates unique crime scenarios, character profiles, clues, and testimonies — all in **Egyptian Arabic dialect (العامية المصرية)**.

## Overview

**The Crime** is a social deduction game for 2–8 players. One (or more) players are secretly assigned as criminals, and the rest are innocents. Each round, players receive partial clues, share testimonies, discuss, and vote to eliminate who they think the criminal is.

The twist: **every game scenario is uniquely generated by AI**, so no two games are alike.

## Prompt Engineering Deep Dive

This is where the magic happens. The AI prompt is the core of the game — it determines the quality, fairness, and fun factor of every game.

### The Master Timeline Architecture (Chain of Thought)

The prompt uses a **4-step Chain of Thought** approach to ensure coherent, interconnected, and fair game scenarios. Instead of asking the AI to generate clues in isolation, we force it to first build a complete hidden story, then extract everything from it.

```
┌─────────────────────────────────────────────────────────┐
│  Step 1: Build Master Timeline (تسلسل الجريمة الكامل)   │
│  6-10 chronological events, each stating:               │
│  - Where EVERY player was                               │
│  - Who they were with                                   │
│  - What they could see/hear from their position         │
│                                                         │
│  This is the "source of truth" — nothing is invented    │
│  outside of this timeline.                              │
└──────────────────────┬──────────────────────────────────┘
                       │ Extract from timeline
                       ▼
┌─────────────────────────────────────────────────────────┐
│  Step 2: Interconnected Extraction (ترابط الأدلة)        │
│  - If A and B were together → both testimonies confirm  │
│  - Clues only contain what a player could observe       │
│    from their actual position in the timeline           │
│  - No inventing evidence outside the timeline           │
└──────────────────────┬──────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────┐
│  Step 3: No Smoking Guns (ممنوع الأدلة المباشرة)         │
│  ❌ No one sees the actual murder                       │
│  ❌ No one finds the weapon in the criminal's hand      │
│  ❌ No single clue reveals the criminal                 │
│  ✅ Only circumstantial evidence:                       │
│     - Hearing an argument through a wall                │
│     - Seeing someone running nervously                  │
│     - Noticing a missing item or moved furniture        │
│     - Smelling something unusual                        │
└──────────────────────┬──────────────────────────────────┘
                       │
                       ▼
┌─────────────────────────────────────────────────────────┐
│  Step 4: First-Person Clue Rule (قاعدة الرؤية الشخصية)  │
│  Every clue written as: "I saw...", "I heard..."        │
│  ❌ Player NEVER gets a clue about themselves           │
│  If Sara saw Ahmed → clue goes to SARA, not Ahmed       │
│  Ahmed CANNOT receive "Sara saw you..."                 │
└─────────────────────────────────────────────────────────┘
```

### Why Master Timeline?

The previous approach asked the AI to generate clues independently for each player, which led to three problems:

| Problem                     | Old Approach                                      | Master Timeline Solution                            |
| --------------------------- | ------------------------------------------------- | --------------------------------------------------- |
| **Disjointed clues**        | Clues contradicted each other or made no sense    | All clues extracted from one coherent timeline      |
| **Smoking gun clues**       | AI sometimes gave direct "I saw the murder" clues | Explicit ban + only circumstantial evidence allowed |
| **Third-person self-clues** | Player gets "You were seen by..."                 | Strict first-person rule + pre-send validation test |

### The Two-Prompt Architecture

The system uses a **System Prompt + User Prompt** pattern sent together to Gemini:

#### 1. System Prompt (`SYSTEM_PROMPT`) — The Game Master Rules

This is a long, carefully crafted prompt (~5000 words in Egyptian Arabic) structured as 4 mandatory steps:

**Step 1 — Language Constraint**

```
كل الكلام لازم يبقى بالعامية المصرية (اللهجة المصرية). متستخدمش فصحى أبداً.
اكتب زي ما المصريين بيتكلموا في حياتهم اليومية.
```

The AI is forced to write everything in Egyptian dialect — not Modern Standard Arabic. This makes the game feel natural and immersive.

**Step 2 — Master Timeline Construction**

The AI must first generate a `master_timeline` — a chronological array of 6-10 events, each containing:

- The event description
- A `player_positions` map showing where every player was, what they were doing, who they were with, and what they could observe

Example event:

```
"لما النور قطع: كريم ومحمد كانوا في الصالة. سارة كانت على السلم —
من مكانها كانت تقدر تسمع خطوات سريعة من ناحية المطبخ."
```

**Step 3 — Interconnected Extraction**

All testimonies and clues are pulled strictly from the timeline:

- **Mutual Confirmation**: If two players were together, both testimonies must mention each other
- **Cross-Angle Observation**: If two players saw the same event from different locations, each describes their own angle
- **No Invention**: No clue can reference something the player couldn't observe from their timeline position

**Step 4 — No Smoking Guns**

Strictly forbidden:

- Seeing the actual murder or violence
- Finding the weapon in someone's hand
- Hearing a direct confession
- Any single clue that solves the mystery alone

Allowed (circumstantial only):

- Hearing an argument through a wall
- Seeing someone running nervously
- Noticing displaced furniture or missing items
- Sensing unusual smells
- Seeing someone enter a room before a power outage

**Step 5 — First-Person Clue Rule**

Every clue written in first person ("I saw...", "I heard..."). The prompt includes a 3-point validation test the AI must run before outputting:

1. Does the clue mention the receiving player's name? → Move it to another player
2. Does the clue say "someone saw you"? → Flip to the observer's perspective
3. Does the clue talk about the player from outside? → Rewrite in first person

**Step 6 — Suspect Balance & Evidence Progression**

Same as before but now grounded in the timeline:

- At least 3 suspects with strong motives
- Evidence difficulty: Round 1 (vague) → Round 4 (specific but needs combining)
- No player can solve alone — cooperation is required

#### 2. User Prompt — The Specific Game Request

Built dynamically and reinforces all 4 steps:

```
Step 1: Build master_timeline (6-10 events with all player positions)
Step 2: Extract all testimonies/clues from timeline only
Step 3: No smoking guns — circumstantial evidence only
Step 4: First-person clues only — no third-person self-references
```

### Prompt Engineering Techniques Used

| Technique                         | How It's Applied                                                                 |
| --------------------------------- | -------------------------------------------------------------------------------- |
| **Chain of Thought (CoT)**        | Force AI to build master_timeline BEFORE generating any clues                    |
| **Role Assignment**               | "إنت مدير لعبة لغز الجريمة" — clear persona                                      |
| **Language Pinning**              | Explicit dialect instructions with correct vs. incorrect word examples           |
| **Negative Examples**             | ❌/✅ pairs for smoking guns, third-person clues, evidence distribution          |
| **Constraint Repetition**         | Critical rules in both system prompt AND user prompt                             |
| **Structured Output**             | Exact JSON schema with `responseMimeType: application/json`                      |
| **Progressive Disclosure**        | Clue difficulty curves explicitly defined per round                              |
| **Pre-Send Validation Checklist** | 10-point checklist AI must verify before responding                              |
| **Grounded Generation**           | All outputs must trace back to the master_timeline — no "hallucinated" clues     |
| **Mutual Consistency**            | If A saw B, then A's testimony mentions B and B's testimony confirms co-location |
| **Temperature Tuning**            | `temperature: 0.9` for creative variety within structured constraints            |

### AI Configuration

```yaml
model: gemini-2.5-flash
temperature: 0.9 # High creativity for unique scenarios
topK: 40 # Broad token sampling
topP: 0.95 # Nucleus sampling
maxOutputTokens: 65536 # Large enough for master_timeline + 8 player packages
responseMimeType: application/json # Force JSON output
```

## AI Response Parsing

After Gemini returns the JSON, the backend:

1. **Extracts JSON** from potential markdown code blocks
2. **Stores `master_timeline`** — the hidden chain-of-thought (never shown to players, kept for validation/debugging)
3. **Parses** the structured response into game objects
4. **Assigns roles** — criminal or innocent per AI's decision
5. **Distributes packages** — each player gets their private info:
   - Character description (grounded in timeline)
   - Mandatory testimony (extracted from timeline — what they saw/heard)
   - Private clues (circumstantial, revealed one per round)
   - Criminal extras: cover story, physical state, tactical note
   - Innocent extras: location, blind spot
6. **Stores ground truth** — only revealed when the game ends

### Criminal vs. Innocent Information Flow

```
     AI builds master_timeline first
              │
              ▼
     Extracts clues from timeline
              │
    ┌─────────┴─────────┐
    ▼                   ▼
 INNOCENT            CRIMINAL
 Gets real clues     Gets misleading tactical advice
 (circumstantial,    ("Create a false suspicion
  first-person,       against an innocent player")
  from timeline)
                     Knows fellow criminals
 Can piece together  Must maintain cover story
 truth with others   Testimony has a hidden flaw
```

## Win Conditions

**Innocents win** when all criminals are eliminated.

**Criminals win** when:

- Active innocents ≤ active criminals (criminals equal or outnumber innocents), **or**
- Maximum rounds are reached without catching all criminals

## How to Run

### Prerequisites

- Java 17+
- Node.js 18+
- A [Google Gemini API key](https://makersuite.google.com/app/apikey)

### 1. Configure the API Key

Open `backend/src/main/resources/application.yml` and set:

```yaml
gemini:
  api-key: YOUR_GEMINI_API_KEY_HERE
  model: gemini-2.5-flash
```

> ⚠️ **See [API Key Security](#api-key-security) before pushing to GitHub.**

### 2. Start the Backend

```bash
cd backend
./mvnw spring-boot:run
```

The backend runs on **http://localhost:8080**.

> If port 8080 is already in use: `kill $(lsof -ti:8080)` then retry.

### 3. Start the Frontend

```bash
cd frontend
npm install
npm run dev
```

The frontend runs on **http://localhost:3000**.

### 4. Play

1. Open **http://localhost:3000**
2. Create a room (set your name, optional setting, criminal count 1–3)
3. Share the 6-character room code with friends
4. Friends join using the code
5. Host starts the game — AI generates the scenario (~10–20 seconds)
6. Read the crime briefing together
7. Check your private role card — you'll see your character, testimony, and first clue
8. Each round: get a new clue → discuss out loud → vote
9. Eliminate suspects until the criminal is caught or the criminals win!

## API Key Security

The API key is currently stored in `application.yml`. **Do not commit your real key to a public repository.**

**Option A — Environment variable (recommended):**

```yaml
# application.yml
gemini:
  api-key: ${GEMINI_API_KEY}
```

```bash
export GEMINI_API_KEY=your_key_here
mvn spring-boot:run
```

**Option B — Git-ignored local override file:**

Create `backend/src/main/resources/application-local.yml` (already in `.gitignore`):

```yaml
gemini:
  api-key: your_key_here
```

```bash
mvn spring-boot:run -Dspring.profiles.active=local
```
